== Context and Scope

=== Business Context

Magrathea ObjectStore provides an S3-compatible REST API. Users interact with it via HTTP to store and retrieve objects in buckets.

.System Context — Magrathea ObjectStore
image::../../c4/images/SystemContext.png[System Context, width=600]

The system context shows:
- **User**: end user interacting via HTTPS — S3-compatible API
- **Magrathea ObjectStore**: Spring Boot 4 reactive S3-compatible object storage

=== Technical Context

| Communication Partner | Input | Output | Protocol |
|----------------------|-------|--------|----------|
| User (AWS CLI, SDK, HTTP client) | S3 REST API requests (XML/JSON) including analytics (`?analytics`), inventory (`?inventory`), metrics (`?metrics`), intelligent-tiering (`?intelligent-tiering`), ABAC (`?abac`), object lock (`?object-lock`), metadata config (`?metadata-config`), metadata table config (`?metadata-table-config`), session (`?session`), directory bucket (`?directory-buckets`), torrent (`?torrent`), restore (`?restore`), select (`?select`), rename (`?rename`), legal hold (`?legal-hold`), retention (`?retention`) queries | S3 REST API responses (XML/JSON) | HTTPS |
| User (binary upload) | Binary data via HTTP PUT | — | HTTP body |
| User (binary download) | — | Binary data via HTTP GET | HTTP body |
