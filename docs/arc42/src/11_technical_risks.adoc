[[section-technical-risks]]
== Risks and Technical Debts

| # | Risk | Impact | Mitigation |
|---|---|---|---|
| 1 | **Single-node in-memory only** — reactive in-memory repositories do not persist to disk | Data loss on restart | Introduce file persistence or R2DBC; `persistence-context-*` placeholder modules ready |
| 2 | **Limited S3 coverage** — 30/111 operations implemented | Unsupported AWS CLI/SDK features | Phases C (Multipart), D (Bucket Config), E (Analytics), F (Advanced) planned in PLAN.md |
| 3 | **In-memory content store** — large objects stored in ConcurrentHashMap | OutOfMemoryError for files > 1GB | Flush to file system, chunked streaming, content store with file persistence |
| 4 | **No authentication/authorization** — AWS SigV4 not implemented | Usable only in trusted environments | Implement SigV4 verification middleware |
| 5 | **No persistence-context integration** — placeholder modules are empty | Alternative persistence untestable | Not a v1 priority — planned as a separate bounded context |
