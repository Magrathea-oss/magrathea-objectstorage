[[section-technical-risks]]
== Risks and Technical Debts

| # | Risk | Impact | Mitigation |
|---|---|---|---|
| 1 | **Single-node in-memory only** — `InMemoryObjectRepository` non persiste su disco | Perdita dati al riavvio | Introdurre file persistence o R2DBC; `persistence-context-*` moduli placeholder pronti |
| 2 | **Limited S3 coverage** — 30/111 operations implementate | AWS CLI/SDK features non supportate | Fasi C (Multipart), D (Bucket Config), E (Analytics), F (Advanced) pianificate in PLAN.md |
| 3 | **In-memory content store** — oggetti grandi in ConcurrentHashMap | OutOfMemoryError per file > 1GB | Flush su file system, chunked streaming, content store con file persistence |
| 4 | **No authentication/authorization** — AWS SigV4 non implementato | Uso solo in ambienti trusted | Implementare SigV4 verification middleware |
| 5 | **No persistence-context integration** — moduli placeholder vuoti | Persistenza alternativa non testabile | Non priorità v1 — pianificato come bounded context separato |
