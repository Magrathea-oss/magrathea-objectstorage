[[section-technical-risks]]
== Risks and Technical Debts

| # | Risk | Impact | Mitigation |
|---|---|---|---|
| 1 | **Default backend is in-memory** — reactive in-memory repositories do not persist to disk | Data loss on restart | Continue storage-engine integration through `storage-engine-reactive-*` modules and verify end-to-end read/write before production claims |
| 2 | **S3 semantic depth** — 111/111 operations are routed as a surface inventory, but route count is not semantic completion. AWS CLI parity has improved for the canonical object CRUD subset, but full S3 scenario parity is incomplete. | AWS SDK features may expect deeper semantics than the in-memory implementation provides | Track semantic gaps in PLAN.md and ADR follow-ups; prioritize AWS CLI/SDK compatibility cases and keep unsupported cases explicitly tagged |
| 3 | **In-memory content store** — large objects stored in ConcurrentHashMap | OutOfMemoryError for files > 1GB | Flush to file system, chunked streaming, content store with file persistence |
| 4 | **No authentication/authorization** — AWS SigV4 not implemented | Usable only in trusted environments | Implement SigV4 verification middleware |
| 5 | **Storage-engine runtime verification incomplete** — policy/YAML/device planning exists, but selected-backend read/write and physical EC shard placement are not yet verified | Alternative persistence may be incorrectly assumed complete | Keep runtime claims gated by Phase 6/7 tests and document `MINIO_STANDARD` as planning-verified only until end-to-end scenarios pass |
| 6 | **Missing checksum validation** — `x-amz-checksum-crc64nvme` and `x-amz-sdk-checksum-algorithm` headers are accepted by the first AWS CLI object CRUD increment, but are not validated. AWS CLI v2 sends these by default and computes the hash automatically. The server should re-compute and verify. | Corrupted payloads accepted silently — data integrity not verified | Read `x-amz-sdk-checksum-algorithm`, re-compute hash with that algorithm, compare with `x-amz-checksum-crc64nvme`; return `400 InvalidArgument` on mismatch |
