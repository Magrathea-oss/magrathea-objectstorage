== Deployment View

=== Development Environment

**Singolo nodo, eseguito localmente.**

| Element | Description |
|---|---|
| S3 Server | JAR `bootstrap-application-1.0.0-SNAPSHOT.jar` in esecuzione su `localhost:8080` |
| File Store | In-memory (ConcurrentHashMap), allocato nello stesso heap JVM del server |
| Java Runtime | JDK 21 |
| Build Tool | Apache Maven 3.9+ |

=== Docker Deployment

Un `Dockerfile` e `docker-compose.yml` sono presenti alla root del progetto per esecuzione containerizzata:

**Dockerfile:**
- Base: `eclipse-temurin:21-jdk`
- Copia `bootstrap-application/target/*.jar`
- Espone porta 8080
- Entrypoint: `java -jar bootstrap-application-1.0.0-SNAPSHOT.jar`

**docker-compose.yml:**
- Singolo servizio `magrathea-s3` su porta `8080:8080`
- Environment variables per configurazione

=== Production Deployment (Future)

L'architettura a due container (`s3-server` + `file-store`) è volutamente monolitica per single-node. In futuro:

- `file-store` può essere sostituito da R2DBC + PostgreSQL per persistenza relazionale
- `file-store` può diventare container separato per scaling orizzontale
- `s3-server` può essere clusterizzato con session affinity
- `persistence-context-*` moduli placeholder per bounded context di persistenza separato
