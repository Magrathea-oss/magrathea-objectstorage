[[section-glossary]]
== Glossary

| Term | Definition |
|---|---|
| Bucket | S3 object container. Identified by a unique DNS-compliant name (3-63 characters, lowercase) |
| S3Object | S3 object: key, metadata, version, ACL, tagging, attributes, binary content |
| ObjectKey | Unique key of an object within a bucket |
| Region | AWS S3 region (e.g., `us-east-1`, `eu-west-1`) |
| StorageClass | S3 storage class: `STANDARD`, `INTELLIGENT_TIERING`, `PARANOIC_MODE` |
| ETag | Entity tag for optimistic concurrency and content verification |
| VersionId | Version identifier for versioned objects |
| ACL | Access Control List: `private`, `public-read`, `public-read-write`, `authenticated-read` |
| Tagging | Key-value pairs for object/bucket categorization |
| CORS | Cross-Origin Resource Sharing: allowed origins, methods, headers |
| Multipart Upload | Upload of large objects in parts (Phase C) |
| RouterFunction | Functional WebFlux pattern for defining HTTP routes programmatically |
| CompletableFuture | Standard Java async — bridge between pure domain and reactive infrastructure |
| Flux<DataBuffer> | Reactive type for binary streaming in WebFlux |
| S3ObjectWrite | Domain interface: object save contract without reactive types |
| S3ObjectContent | Domain interface: object read contract without reactive types |
| ARC42 | Architecture documentation template in 12 sections |
| ADR | Architecture Decision Record (Michael Nygard format) |
| C4 Model | Architecture description method with 4 levels (Context, Container, Component, Code) |
| Structurizr DSL | Domain-specific language for modeling C4 diagrams |
