== Introduction and Goals

The Magrathea ObjectStore project implements an **AWS S3-compatible object storage** built with Spring Boot 4 WebFlux and Java 21.

It provides a lightweight, single-node implementation of the AWS S3 REST API for development, testing, and small-scale deployments.

=== Goals

- **S3 API compatibility**: implement the in-scope AWS S3 REST API operation set (111/111), including Phase F advanced/specialized operations
- **Reactive by design**: full WebFlux non-blocking I/O, DataBuffer streaming for binary content
- **Domain-Driven Design**: pure domain with zero framework dependencies, hexagonal architecture
- **Testability**: 3-layer testing (JUnit domain, Cucumber BDD integration, AWS CLI compatibility)

=== Stakeholders

| Role | Contact | Expectations |
|------|---------|-------------|
| Developer | Development team | Clean DDD code, reactive patterns, comprehensive tests |
| Tester | QA team | Cucumber BDD scenarios, AWS CLI compatibility verification |
| Architect | Architecture team | ARC42 documentation, ADRs, C4 diagrams, DDD compliance |

=== Motivation

The project serves as both a production-quality S3-compatible storage and a reference implementation of Java DDD + Spring Boot 4 reactive patterns with hexagonal architecture.
